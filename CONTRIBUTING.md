# Contributing Guidelines

## Branching strategy

- All work must be done on feature branches created from `dev`.
- Branch names should follow the pattern: `feature/issue-XX-short-description`.

## Commit rules

- All commits must be signed.
- Commit messages should be clear and descriptive.

## Pull Requests

- Open a Pull Request from your feature branch to `dev`.
- Ensure all checks pass before requesting a review.
- Reference the related issue in the PR description (e.g. `Closes #8`).

## Code style

- Follow the project conventions and existing patterns.
- Run available checks or scripts before pushing when applicable.
